# CalendarApp (Backend)

Бэкенд-часть приложения-календаря, построенная на базе веб-технологий .NET 8. Проект реализует REST API для управления задачами, пользователями и группами с сохранением данных в реляционной базе данных PostgreSQL.

## 🛠 Стек технологий

* **Платформа:** .NET 8 (Web API)
* **ORM:** Entity Framework Core 8 (Code-First approach)
* **База данных:** PostgreSQL
* **Контейнеризация:** Docker
* **Безопасность:** BCrypt.Net-Next (хэширование паролей с солью), настроенная политика CORS
* **Документация API:** Swagger / OpenAPI

---

## 🗄 Структура базы данных и архитектура

Схема данных автоматически генерируется через миграции EF Core и включает в себя следующие сущности:
* `Tasks` — задачи (название, описание, временные метки начала/окончания, признак внешнего события).
* `Groups` — категории/календари задач с цветовым оформлением (`ColorHex` для фронтенда).
* `Users` — пользователи системы (с безопасным хранением `PasswordHash`).
* Связь **Many-to-Many** между задачами и пользователями (реализована через связующую таблицу `TaskItemUser` для назначения нескольких исполнителей на одну задачу).

> 💡 **Архитектурная нота:** Взаимодействие с API полностью переведено на **DTO (Data Transfer Objects)**. Это исключает циклические зависимости при передаче связанных сущностей и гарантирует, что конфиденциальные данные (хэши паролей) никогда не утекут на клиентскую часть.

## 🚀 Как запустить проект локально

### 1. Предварительные требования
* Установленный **.NET 8 SDK** в среде Linux/WSL.
* Запущенный и настроенный контейнер **PostgreSQL в Docker**.

Если контейнер еще не создан, его можно запустить следующей командой:

```bash
docker run --name calendar-postgres -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=your_secure_password -e POSTGRES_DB=calendar_db -p 5432:5432 -d postgres
```

(При необходимости актуализируйте строку подключения ConnectionStrings:DefaultConnection в файле appsettings.json).

2. Применение миграций к базе данных
Если база данных запускается впервые, необходимо применить миграции для автоматического создания всех таблиц и индексов.

Находясь в папке проекта CalendarApp.Api, выполните команду через прямой путь к инструментам .NET:

```bash
~/.dotnet/tools/dotnet-ef database update
```
3. Запуск веб-сервера
Перейдите в директорию API-приложения и запустите сервер:

```bash
dotnet run
```
После успешного запуска веб-сервер начнет прослушивать порт. Интерактивная документация Swagger UI для отправки тестовых запросов будет доступна в браузере по адресу:
👉 http://localhost:5171/swagger
